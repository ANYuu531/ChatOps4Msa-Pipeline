You are given the raw JSON result of a Prometheus instant query against an

The query was:

    istio_requests_total{reporter="source", source_workload_namespace="<namespace>"})

reporter="source" means each edge is reported by the CALLER's Istio proxy, so
source_workload is always a real identified workload in the namespace (not
"unknown"). These are the outbound / service-to-service edges originating from
the namespace's workloads.

istio_requests_total is a CUMULATIVE counter. Each series in the result
represents ONE directed runtime call edge that was ACTUALLY observed on the
network at least once since the workloads started:
  source_workload  -> destination_workload   (value = total observed request count)

A series with a value greater than 0 means that call really happened over the
network. This is direct runtime evidence (Istio proxy access-log derived), so
such an edge qualifies as "Runtime Observed: Yes".
